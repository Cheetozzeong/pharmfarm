# 팜팜 약사 사용 가이드

팜팜은 앱과 관리자 페이지를 함께 사용합니다. 앱에서는 입고와 반품을 처리하고, 관리자 페이지에서는 재고, 처방 차감 결과, 초과 처방, 구매 내역을 확인합니다.

## 전체 흐름

- 앱에서 입고 처리한 약품은 관리자 페이지의 재고에서 확인합니다.
- 앱에서 반품할 약품을 스캔하면 입고 이력 또는 구매 내역을 기준으로 판매처를 확인합니다.
- 처방전 차감 결과와 재고 부족으로 생긴 초과 처방은 관리자 페이지에서 확인하고 처리합니다.

## 앱 사용법

### 1. 입고

- 앱에서 `입고`를 선택합니다.
- 먼저 `도매처 선택` 화면에서 입고할 도매처를 선택합니다.
- 약품 QR을 스캔합니다.
- `입고 확인` 화면에서 약품명, 수량, 유효기간을 확인합니다.
- 약품 후보를 선택해야 하는 항목이 있으면 실제 입고할 약품을 선택합니다.
- 문제가 없으면 `입고 확정`을 눌러 입고를 완료합니다.
- 완료 후 계속 스캔하거나 관리자 페이지의 재고에서 반영 결과를 확인합니다.

입고 결과의 매칭 상태는 다음 기준으로 확인합니다.

- `정상매칭`: QR 정보로 약품이 바로 확인된 상태입니다. 약품명, 수량, 유효기간만 확인하고 입고 확정하면 됩니다.
- `이름매칭`: 약품명으로 후보를 찾은 상태입니다. 후보 목록에서 실제 입고할 약품을 선택한 뒤 입고 확정합니다.
- `미등록` 또는 `미지정`: 기준 데이터에서 약품을 바로 찾지 못한 상태입니다. 임의 약품명과 보험코드를 확인하거나 입력한 뒤 처리합니다. 확실하지 않으면 입고 확정 전에 담당자에게 확인합니다.
- `임의`: 기준 데이터와 연결하지 않고 임의 재고로 등록되는 상태입니다. 실제 약품명과 보험코드가 맞는지 확인한 뒤 입고 확정합니다.

### 2. 반품

- 앱에서 `반품`을 선택합니다.
- 반품할 약품의 QR 또는 바코드를 스캔합니다.
- 입고 이력이 있으면 확정 도매처와 반품 가능 수량이 표시됩니다.
- 입고 이력이 없으면 구매 내역 기준 판매처 후보가 표시될 수 있습니다.
- 판매처와 반품 수량을 확인한 뒤 반품을 확정합니다.
- 입고 이력이나 판매처 후보가 없으면 반품 대상으로 등록하지 않습니다.

### 3. 스캔 시 확인할 내용

- 이미 처리된 약품 안내가 나오면 같은 QR을 다시 처리하지 않습니다.
- 유효기간 확인 안내가 나오면 약통의 유효기간을 확인합니다.
- 인식이 잘 안 되면 QR 또는 바코드를 화면 중앙에 맞추고 다시 스캔합니다.

## 관리자 페이지 사용법

### 1. 대시보드

로그인 후 처음 보이는 화면입니다.

- 오늘 처리할 일을 먼저 확인합니다.
- 처방 차감 실패, 초과 처방, 재고 부족, 임의 재고 같은 확인이 필요한 항목이 표시됩니다.
- 금액 관련 정보는 권한이 있는 계정에서만 보일 수 있습니다.

### 2. 재고

현재 약국에 등록된 재고를 확인하는 화면입니다.

- 약품명 또는 보험코드로 검색할 수 있습니다.
- 전체, 향정, 비향정, 임의 재고를 구분해서 볼 수 있습니다.
- 재고 수량, 보험코드, 약품명, 금액 정보를 확인합니다.
- 필요한 경우 약품을 직접 등록하거나 재고 수량을 조정할 수 있습니다.

### 3. 초과 처방

처방 수량보다 보유 재고가 부족했던 약품을 확인하는 화면입니다.

- `주문 필요`: 아직 주문 처리가 필요한 항목입니다.
- `주문 완료`: 부족분에 대해 주문을 넣은 항목입니다.
- `보류`: 당장 처리하지 않고 따로 확인할 항목입니다.

목록에서 항목을 선택하면 해당 약품의 부족 수량과 관련 처방전 전체 내용을 확인할 수 있습니다. 상태는 필요하면 다시 변경할 수 있습니다.

### 4. 처방전

처방전 차감 결과를 확인하는 화면입니다.

- 자동 차감된 처방과 수동 처리가 필요한 처방을 확인합니다.
- 차감 실패 항목은 기존 재고와 연결하거나, 임의 약품으로 처리하거나, 재고 반영 없이 완료 처리할 수 있습니다.
- 기존 재고로 처리할 때 재고가 부족하면 부족분은 초과 처방으로 넘어갑니다.

### 5. 구매 내역

구매 내역을 불러오고 확인하는 화면입니다.

- 안내받은 `sessionid`, `csrftoken` 값을 입력해 구매 내역 연결 정보를 저장합니다.
- 시작일과 종료일을 선택한 뒤 구매 내역 불러오기를 실행합니다.
- 불러온 구매 내역은 반품 시 판매처를 확인하는 참고 자료로 사용됩니다.

### 6. 계정에 따라 보이지 않는 메뉴

일부 메뉴는 운영자용 계정에서만 보일 수 있습니다.

- 기준 데이터
- Import
- 도매처 관리
- 계정 생성

일반 약국 사용자는 앱의 입고와 반품, 관리자 페이지의 재고, 처방전, 초과 처방, 구매 내역 중심으로 사용하면 됩니다.
